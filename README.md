## Описание проекта
Данный проект представляет собой полноценный конвейер для обучения и развертывания модели распознавания ключевых слов (Keyword Spotting, KWS) в реальном времени. Система классифицирует аудиопоток с микрофона на 12 классов: 10 целевых слов (`yes`, `no`, `up`, `down`, `left`, `right`, `on`, `off`, `stop`, `go`), а также классы `unknown` (неизвестное слово) и `silence` (тишина/фоновый шум).

Архитектура решения включает:
- Сверточную нейронную сеть (CNN) для классификации мел-спектрограмм.
- Модуль обучения с аугментацией данных, сохранением контрольных точек (checkpoints) и логированием метрик.
- Веб-интерфейс, использующий `AudioWorklet` для захвата звука и `WebSocket` для потоковой передачи аудиоданных на бэкенд с минимальной задержкой.

## Хранение артефактов
Все артефакты модели хранятся в директории `models/` (необходимо создать её вручную перед началом работы):
- `models/kws_model.pth` – веса лучшей обученной модели.
- `models/checkpoint.pth` – контрольная точка для возобновления обучения (веса модели, оптимизатора, планировщика).
- `models/training_stats.pkl` – история метрик обучения (loss, accuracy по эпохам).

## Обучение модели
Для запуска процесса обучения выполните:
```bash
python train.py
```
Скрипт автоматически:
- Загружает тренировочную и валидационную выборки.
- Применяет аугментацию (временной сдвиг, изменение громкости, добавление фонового шума) только к тренировочным данным.
- Использует планировщик `ReduceLROnPlateau` для адаптивного изменения скорости обучения.
- Сохраняет лучший результат в `models/kws_model.pth` и чекпоинт для возобновления в `models/checkpoint.pth`.

## Запуск приложения
Для одновременного запуска веб-сервера (фронтенд) и WebSocket-сервера (бэкенд) выполните:
```bash
python run.py
```
- Фронтенд будет доступен по адресу: `http://localhost:8000`
- Бэкенд будет принимать подключения по адресу: `ws://localhost:8765`

После запуска откройте `http://localhost:8000` в браузере, предоставьте разрешение на использование микрофона и нажмите кнопку **Start Listening**.

## Принцип работы в реальном времени
1. Браузер запрашивает доступ к микрофону.
2. `AudioWorklet` разбивает аудиопоток на фрагменты, преобразует их в формат `Int16Array` и передает в основной поток.
3. Основной поток отправляет бинарные данные через WebSocket на бэкенд.
4. `AudioProcessor` на бэкенде накапливает данные в буфер, извлекает мел-спектрограмму размером `[1, 40, 100]` и передает её в модель.
5. При превышении порога уверенности (`CONFIDENCE_THRESHOLD = 0.75`) и с учетом задержки (`cooldown`) сервер отправляет клиенту сообщение формата `DETECTED:<слово>`, которое отображается в интерфейсе.

## Настройка параметров
Основные параметры системы вынесены в файл `config.py`:
- `TARGET_SAMPLE_RATE`: 16000 Гц.
- `CHUNK_DURATION`: 0.064 сек (длина одного фрагмента).
- `CONFIDENCE_THRESHOLD`: 0.75 (минимальная уверенность для детекции).
- `MODEL_PATH`: путь к файлу весов модели.
